API & Routes

These are the main routes and API endpoints for SCD Reporting. All routes require authentication unless marked Public. For API routes, send Authorization: Bearer <token> header with your API token. Generate or rotate your token at Profile. For more information on using the API, see the documentation and scripts in the project GitHub repository.

REST API

Send Content-Type: application/json and Authorization: Bearer <token>. Generate your token at Profile.

POST
/api/entries/ Login

Create a new work entry. Returns 201 on success or 400 with an errors object on validation failure.

Request body

{
  "title":        "string (required)",
  "description":  "string (required)",
  "project":      "slug or id (required)",
  "category":     "slug or id (required)",
  "period_kind":  "today|week|fortnight|month|custom",
  "period_start": "YYYY-MM-DD (required)",
  "period_end":   "YYYY-MM-DD (required)",
  "group":        "slug or id (optional)",
  "lab_priority": "slug or id (optional)",
  "tags":         ["tag1", "tag2"],
  "is_private":           false,
  "is_critical":          false,
  "is_highlight":         false,
  "highlight_stars":      0,
  "is_division_head_only": false
}

Response (201)

{
  "id":           42,
  "title":        "Weekly status",
  "project":      "project-slug",
  "category":     "category-slug",
  "period_kind":  "week",
  "period_start": "2026-05-26",
  "period_end":   "2026-06-01",
  "url":          "https://…/entries/42/"
}

Response (400)

{
  "errors": {
    "title": "This field is required.",
    "project": "Project \"bad-slug\" not found."
  }
}

Example — curl

curl -s -X POST https://scd-reporting.fnal.gov/api/entries/ \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-api-token>" \
  -d '{
    "title": "Weekly status update",
    "description": "Project Milestone:\nPerformance Goal: FY26-Q3\nUpdate: Completed integration tests.",
    "project": "mu2e-daq",
    "category": "software-development",
    "period_kind": "week",
    "period_start": "2026-05-26",
    "period_end": "2026-06-01"
  }'

Example — Python (requests)

import requests

TOKEN = "<your-api-token>"
URL   = "https://scd-reporting.fnal.gov/api/entries/"

payload = {
    "title":        "Weekly status update",
    "description":  "Project Milestone:\nPerformance Goal: FY26-Q3\nUpdate: Completed integration tests.",
    "project":      "mu2e-daq",
    "category":     "software-development",
    "period_kind":  "week",
    "period_start": "2026-05-26",
    "period_end":   "2026-06-01",
}

r = requests.post(URL, json=payload, headers={"Authorization": f"Bearer {TOKEN}"})
r.raise_for_status()
data = r.json()
print(f"Created entry #{data['id']}: {data['url']}")

Example — bash (from JSON file)

TOKEN="<your-api-token>"

# post from a JSON file
curl -s -X POST https://scd-reporting.fnal.gov/api/entries/ \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ${TOKEN}" \
  -d @entry.json

# or post inline from a variable
DATA='{"title":"Weekly status","description":"...","project":"mu2e-daq","category":"software-development","period_kind":"week","period_start":"2026-05-26","period_end":"2026-06-01"}'
curl -s -X POST https://scd-reporting.fnal.gov/api/entries/ \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ${TOKEN}" \
  -d "$DATA"

Standalone scripts

scripts/scd-post-entry-json (Python) and scripts/scd-post-entry-json.sh (bash) read a JSON file (or stdin) and post a single entry. Both read ~/.config/scd-reporting/config.yaml for url and token, overridable via SCD_URL / SCD_TOKEN env vars or -u / -t flags.
